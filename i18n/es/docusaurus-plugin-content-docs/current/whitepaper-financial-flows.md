## 5.1 Flujo de Compra
1. El cliente presenta una orden de compra.
2. Los fondos se reciben a través de cuentas institucionales.
3. Se realizan verificaciones de cumplimiento.
4. Aureum One adquiere oro o asigna del inventario.
5. El custodio confirma la posición de oro.
6. Se actualiza la asignación interna.
7. El token se acuña 1:1 en XRPL.
8. El auditor revisa muestras y eventos críticos.

## 5.2 Flujo de Redención
1. El cliente solicita la redención.
2. El token se quema o se mueve a reserva.
3. El oro se liquida o se ajusta el inventario.
4. El fiat se devuelve al cliente.
5. Se registra el rastro de auditoría completo.
